# Academic diagram style

- Use a white or very light pastel background with clear left-to-right or top-to-bottom
  narrative flow.
- Group stages with low-saturation containers; reserve saturated color for active,
  critical, erroneous, or final elements.
- Use rounded rectangles for processes, grids or cuboids for tensors, and cylinders
  only for databases, buffers, or memory.
- Use solid connectors for primary data flow and dashed connectors for auxiliary,
  optional, gradient, or update flow. Prefer orthogonal lines for architectures and
  smooth curves for feedback loops.
- Use sans-serif text for labels and serif italic text for mathematical variables.
- Keep a consistent geometry system, border weight, arrowhead style, palette, and icon
  family.
- For agent systems, use restrained document, chat, tool, memory, and user-interface
  motifs. For vision/3D, use camera frustums, rays, grids, or point clouds only when
  semantically relevant. For theory, prefer minimal graph and manifold forms.
- Avoid heavy black outlines, saturated zone backgrounds, mixed visual dimensions,
  decorative legends, PowerPoint-default colors, and identical line styles for
  different semantics.
